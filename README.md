# Bible App

聖書アプリケーションのプロジェクトです。バージョンごとに異なるコンセプトで UI/UX が設計されています。

## バージョン更新履歴と特徴

### v5 (PC Pro 版)

デスクトップでの利用に特化した、高機能・高生産性バージョンです。

- **特徴**:
  - **2 カラムレイアウト**: 左側に操作パネルを常駐させ、PC の大画面を有効活用。
  - **クイックジャンプ**: 「ヨハ 3:16」のように入力して瞬時に移動可能。
  - **ショートカットキー**: マウスなしで主要操作が可能（Ctrl+G: 移動, Ctrl+F: 検索, Ctrl+D: ダークモードなど）。
  - **クリップボードコピー**: 聖句ごとのコピーボタンを実装。
- **技術**: 高速な DOM 操作、キーボードイベントリスナー、クリップボード API。

### v4 (Modern Mobile App)

スマホアプリのような操作感を目指した、モダン UI バージョンです。

- **特徴**:
  - **左右サイドバー**: 左に履歴・ブックマーク・検索、右に設定メニューを配置。
  - **ボトムナビゲーション**: 画面下部に「前の章」「次の章」などの操作系を集約。
  - **言語設定**: 日本語・中国語の表示順序切り替え（日中/中日）や、言語ごとのルビ ON/OFF が可能。
  - **PWA 対応**: ホーム画面に追加してアプリとして動作可能。スライドショー付きのお知らせ機能搭載。
- **技術**: localStorage による履歴/ブックマーク管理、Service Worker、Manifest.json。

### v3 (Speed & Input Mode)

番号入力による「爆速検索」に特化した、玄人向けバージョンです。

- **特徴**:
  - **番号入力モード**: 書巻番号（例: 1=創世記, 43=ヨハネ）と章番号を入力するだけで即座に表示。
  - **範囲検索**: 「検索モード([)」と「節範囲指定(])」をキーボードで切り替え可能。
  - **軽量化**: 必要な機能だけを削ぎ落とし、表示速度を最優先。
- **技術**: インデックス化による高速ルックアップ処理。

### v2 (Standard Mobile)

ハンバーガーメニューを採用した、標準的なモバイル対応バージョンです。

- **特徴**:
  - **階層メニュー**: 書巻 → 章 → 節 と順番に選択する確実なナビゲーション。
  - **シンプル検索**: キーワードによる全文検索機能。
  - **基本機能**: 文字サイズ変更、日本語ルビの ON/OFF。
- **技術**: 基本的な DOM 操作、Service Worker によるオフラインキャッシュ（初期実装）。

## 開発環境・Git 設定ガイド（重要）

このリポジトリは、Global 設定とは別のサブアカウント（`tjc-kousei`）で運用しており、大容量ファイルの通信設定も含んでいます。
環境が変わった際やエラーが出た際は、以下の設定を適用してください。

### 1. アカウント名義の設定（ローカル限定）

このリポジトリ内でのみ有効なユーザー名とメールアドレスを設定します。

```bash
git config --local user.name "tjc-kousei"
git config --local user.email "（GitHub登録メールアドレス）"
```

### 2. 認証とリモート URL の設定

パスワード（トークン）入力を省略し、確実にサブアカウントで接続するための設定です。

```bash
# ユーザー名を含んだURLをリモートに設定
git remote set-url origin https://tjc-kousei@github.com/tjc-kousei/bible_app.git

# 認証情報をPCに保存（次回からトークン入力不要）
git config --local credential.helper store
```

### 3. 通信エラー対策（RPC failed / HTTP 400）

「RPC failed」や「The remote end hung up unexpectedly」のエラーが出る場合（大きなファイルを Push する際など）の設定です。

```bash
# 通信バッファサイズを500MBに拡張
git config --local http.postBuffer 524288000

# HTTPバージョンを1.1に固定（通信の安定化）
git config --local http.version HTTP/1.1
```

### 4. 強制 Push が必要な場合

リモート（GitHub）とローカルの履歴が合わず、エラーが出る場合に使用します。

```bash
git push -f origin main
```
